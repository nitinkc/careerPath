# Developer → CTO Career Journey

A comprehensive learning guide for computer science graduates and software engineers who want to understand the corporate hierarchy, career progression, and skills required to reach the Chief Technology Officer (CTO) level.

## What's Included

- **8 Career Levels** mapped to industry standards (Junior → CTO)
- **Skills by Level**: Technical, leadership, and business acumen required at each stage
- **Interview Preparation**: Questions you'll face and how to excel
- **20+ Essential Books** for career growth
- **30-Year Career Roadmap**: Long-term planning from entry-level to executive
- **Decision Points**: When to choose IC (Individual Contributor) vs Management track

## Running Locally

### Prerequisites
- Python 3.8+ installed
- Virtual environment activated (or dependencies installed)

### Setup & Run

```bash
# Install dependencies (if not already done)
pip install -r requirements.txt

# Activate virtual environment (if you have one)
source .venv/bin/activate  # On macOS/Linux
# or
.venv\Scripts\activate     # On Windows

# Start the development server
mkdocs serve

# If port 8000 is already in use, run on an alternate port:
mkdocs serve --dev-addr 127.0.0.1:8001
```

Then open your browser to:
- **Default**: http://127.0.0.1:8000
- **Alternate port**: http://127.0.0.1:8001

The site will auto-reload when you edit files.

## Key Insights

- **The IC track is as valuable as management** — you don't have to manage people to become CTO
- **Business acumen matters more at each level** — technical skills alone plateau at Senior Engineer
- **The first 5 years are foundation** — don't rush; learn deeply
- **Domain expertise is your leverage** — specialize before generalizing
- **Leadership (not management) is required at all levels** — even as an IC
